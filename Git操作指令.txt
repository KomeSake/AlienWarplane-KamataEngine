当前状态：git status

新增全部文件：git add .
新增指定文件：git add 文件名
提交：git commit -m "提交的注释"

远程库操作
拉取库：git pull (origin 分支名)
推送库：git push (origin 分支名)
 *首次push可以加上 -u 来设置和远程库的分支有关联
  这样就不需要输入origin(库名)main(分支名)这么麻烦了
  或者通过git branch -u origin/main来手动关联

分支相关
查看分支：git branch
创建分支：git branch 分支名
移动到此分支：git checkout 分支名
创建并移动到此分支：git checkout -b 分支名

创建Git管理目录
初始化：git init
直接拉取库：git clone 地址 (指定文件目录)
和库建立联系：git remote add origin 地址
(注意！如果项目是私人的，需要通过SSH密钥链接来联系远程库)

生成SSH钥匙
ssh-keygen -t rsa(按3下空格后，会在C/uesr/.ssh中找到)
设置用户名和邮箱
git config --global user.name "YourName"
git config --global user.email "YourEmail@xx.com"
查看当前设置：git config -l

无视指定文件
.gitignore（需要以文件的方式，放在文件夹的根目录）

初次流程(将代码上传给GitHub托管)：
1、GitHub里面创建对应的远程库
	1.1、库中创建好.gitignore和REMADME.md文件
	1.2、.gitignore中写入.vs/可以忽略掉vs的本地生成文件
2、为本地文件初始化Git管理
3、联系远程库
4、将远程库的东西pull下来
5、避免错误，最好用指令把main分支重命名
	5.1、git branch -M main
6、最后就是把本地文件推送上远程库即可
	6.1、git push -u origin main
